# Logger

AzuraJS exposes **two** logging surfaces:

1. **Callable `logger`** and **`log` shortcuts** from **`azurajs/logger`** ‚Äî same style as **v2** (`logger("success", "‚Ä¶")`, icons and colors in the terminal).
2. **`Logger` class** from **`azurajs/utils`** ‚Äî structured **`.info()` / `.error()`** API used by **`AzuraServer`** and middleware.

<Callout type="warning" title="Do not mix up the two `logger` symbols">
  **`import { logger } from "azurajs/logger"`** is a **function**. It is **not** an instance of **`Logger`**, so **`logger.info(...)`** is a TypeScript error. Use **`log.info(...)`** from **`azurajs/logger`**, or **`new Logger()`** from **`azurajs/utils`** for **`.info()`** on an instance.
</Callout>

## Callable logger (`azurajs/logger`)

For startup messages, Redis/DB connection logs, and scripts:

```typescript
import { logger, log, configureLogger } from "azurajs/logger";

logger("log", "Connecting to Redis‚Ä¶");
logger("success", "Connected to Redis.");
logger("fatal", "Could not connect.");

// Same levels via shortcuts:
log.info("Listening on port 3000");
log.error("Something went wrong");
```

### Levels

First argument to **`logger(level, message, options?)`**:

| Level | Typical use |
| :--- | :--- |
| `debug`, `trace` | Verbose diagnostics |
| `info`, `log` | Normal messages |
| `success` | Positive outcome |
| `warn` | Warnings |
| `error`, `fatal` | Written with **`console.error`** |

### `configureLogger`

Override defaults (prefix, timestamps, icons, colors) globally:

```typescript
configureLogger({
  prefix: "MyAPI",
  showTimestamp: true,
  timestampFormat: "iso",
});
```

### `colors` helper

`import { colors } from "azurajs/logger"` ‚Äî **`colors.hex`**, **`colors.rgb`**, **`colors.ansi`**, etc., for custom ANSI output.

## `Logger` class (`azurajs/utils`)

Use when you need an object with methods or when integrating with **`AzuraServer`** (the server constructs its own **`Logger`**).

```typescript
import { Logger } from "azurajs/utils";

const log = new Logger({
  level: "debug",
  prefix: "my-app",
  timestamp: true,
  colors: true,
});

log.debug("Cache miss");
log.info("User %s logged in", userId);
log.warn("Deprecated route");
log.error("DB error:", err.message);
```

### Constructor options

| Option | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `level` | `debug` ‚Ä¶ `silent` | `"info"` | Minimum level to print |
| `prefix` | `string` | `"azura"` | Label on each line |
| `timestamp` | `boolean` | `true` | ISO-style time prefix |
| `colors` | `boolean` | TTY auto | ANSI colors |

### Levels (class)

Severity order: `debug` ‚Üí `info` ‚Üí `warn` ‚Üí `error` ‚Üí `silent` (nothing printed).

### Other methods

- **`setLevel(level)`** ‚Äî change threshold at runtime  
- **`request(method, path, status, durationNs)`** ‚Äî HTTP line (used internally)  
- **`banner(port, host)`** ‚Äî startup banner after **`listen`**

## LoggingMiddleware

From **`azurajs/middleware`**: measures handler time, sets **`X-Response-Time`**, and logs via the **server‚Äôs** **`Logger`** instance.

```typescript
import { AzuraServer } from "azurajs";
import { LoggingMiddleware } from "azurajs/middleware";

const app = new AzuraServer();
app.plugin(LoggingMiddleware());
```

## Choosing an API

| Situation | Use |
| :--- | :--- |
| Quick scripts, parity with **v2** tutorials | **`azurajs/logger`** ‚Äî **`logger("success", ‚Ä¶)`** |
| Need **`.info()`** on a single import | **`log`** from **`azurajs/logger`** or **`new Logger()`** |
| Custom prefix per module | **`new Logger({ prefix: "db" })`** |
| Request timing header + access log | **`LoggingMiddleware`** |
