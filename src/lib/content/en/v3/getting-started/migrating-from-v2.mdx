# Migrating from v2 to v3

AzuraJS v3 is a full rewrite: same goals (fast, TypeScript-first, decorators), but a **new core class**, a plugin model, an **`exports` map on npm** with many subpaths, and some public API differences. This page is the main **v2 → v3** guide: checklist, changes by topic, and the most common post-upgrade issues.

<Callout type="info" title="Versions">
  **v2** means **`azurajs` @ 2.x** and legacy APIs (`AzuraClient`, `setupSwagger`, etc.). **v3** is **`azurajs` ^3.x** with **`AzuraServer`**, plugins, and the subpaths listed in [npm package and exports](/docs/en/v3/reference/package-and-exports).
</Callout>

## Migration checklist

1. **Dependency** — Pin **`azurajs@^3`** in `package.json` and remove any lockfile resolutions pointing at 2.x.
2. **Entry point** — Replace `AzuraClient` with **`AzuraServer`** and review `listen` / server options ([Configuration](/docs/en/v3/getting-started/configuration)).
3. **Plugins** — Move from ad-hoc helpers to **`app.plugin(...)`** factories (`CORSPlugin`, `HelmetPlugin`, …); CORS may still be enabled via `config.plugins.cors` on **`AzuraServer`**.
4. **Swagger** — Replace `setupSwagger` / `setupSwaggerWithControllers` with **`SwaggerPlugin`** and align URLs (`/openapi.json` vs `/api-spec.json`).
5. **Validation** — Keep `import { v } from "azurajs/validators"`; use **`DtoValidationError`** for **DTOValidator** errors (do not confuse with the **`ValidationError`** class from **`v`**).
6. **Logger** — Callable **`logger`** lives in **`azurajs/logger`**; the **`Logger`** class is in **`azurajs/utils`**.
7. **Imports** — Point to the new subpaths from `package.json` (full table in [package reference](/docs/en/v3/reference/package-and-exports)).
8. **QA** — Run tests, verify OpenAPI, decorator routes, and JWT/session flows.

## 1. Package name and install

| v2 | v3 |
| :--- | :--- |
| `azurajs` @ 2.x | **`azurajs` @ ^3.x** (Node **>= 18**) |

```bash
npm install azurajs@^3
```

Do not mix v2 and v3 majors in the same app.

### `package.json` surface in v3

v3 publishes **many subpaths** (`azurajs/core`, `azurajs/plugins`, `azurajs/validators`, `azurajs/swagger`, `azurajs/infra`, …). Full index: [npm package and exports](/docs/en/v3/reference/package-and-exports).

## 2. Application entry: `AzuraClient` → `AzuraServer`

| v2 | v3 |
| :--- | :--- |
| `new AzuraClient()` | `new AzuraServer({ ... })` |

```typescript
// v2-style
// import { AzuraClient } from "azurajs";

// v3
import { AzuraServer } from "azurajs";
```

- **`app.register(MyController)`** for **`@Controller`** classes.
- **`app.get` / `app.post`** remain available for Express-style routing.

See [Configuration](/docs/en/v3/getting-started/configuration) and [Quick start](/docs/en/v3/getting-started/quick-start).

## 3. Subpaths that help v2-era code

| Scenario | v3 subpath |
| :--- | :--- |
| Old code imported “infra” | **`azurajs/infra`** — `ConfigModule`, `resolveIp` |
| Only `Router` | **`azurajs/router`** — `Router` |
| Only `HttpError` | **`azurajs/http-error`** |
| Cookie helpers only | **`azurajs/cookies`** |
| CORS or rate limit without importing all of `plugins` | **`azurajs/cors`**, **`azurajs/rate-limit`** |

This matches what the published `exports` field exposes and keeps bundles smaller.

## 4. Swagger / OpenAPI

| v2 | v3 |
| :--- | :--- |
| `setupSwagger`, `setupSwaggerWithControllers`, spec often at **`/api-spec.json`** | **`SwaggerPlugin`** with **`getDocuments: () => app.getRouteDocuments()`**, default spec **`/openapi.json`**, UI **`/docs`** |
| Older decorator import paths | **`ApiTags`**, **`ApiOperation`** from **`azurajs`** or **`azurajs/swagger`** |

**v3 minimal pattern:**

```typescript
import { AzuraServer, SwaggerPlugin } from "azurajs";

const app = new AzuraServer({ /* ... */ });

app.plugin(
  SwaggerPlugin({
    getDocuments: () => app.getRouteDocuments(),
    info: { title: "My API", version: "1.0.0" },
    specPath: "/openapi.json",
    docsPath: "/docs",
  }),
);
```

**Documentation UI:** v3 ships a **custom HTML UI** (sidebar, Inter / JetBrains Mono). It is **not** Swagger UI from a CDN. Static asset path: **`azurajs/swagger-ui`** → `swagger-ui-modern.html`.

If you relied on **`/api-spec.json`**, set **`specPath: "/api-spec.json"`** in **`SwaggerPlugin`**.

## 5. Validators: `v` and DTO types

| v2 | v3 |
| :--- | :--- |
| `import { v } from "azurajs/validators"` | Same: **`v`**, **`ValidationError`** (class), **`Infer`**, etc. |
| DTO errors typed as **`ValidationError`** | DTO flows use **`DtoValidationError`** so it does not clash with **`ValidationError`** from **`v`** |

**Environment schema example:**

```typescript
import { v } from "azurajs/validators";

const envSchema = v.object({
  PORT: v.string(),
  NODE_ENV: v.string(),
});

export const env = envSchema.parse(process.env);
```

**DTOValidator:**

```typescript
import type { ValidationResult, DtoValidationError } from "azurajs/validators";
```

See [Validation](/docs/en/v3/features/validation).

## 6. Logger: function vs class

| API | Import | Use when |
| :--- | :--- | :--- |
| **Callable `logger`** (v2-style) | **`azurajs/logger`** | `logger("log", "msg")`, `logger("success", "…")`, `logger("fatal", "…")`, or **`log.info(...)`** |
| **`Logger` class** | **`azurajs/utils`** | `new Logger({ level: "info" })`, **`.info()` / `.error()`**, used internally by **`AzuraServer`** |

Do **not** treat **`import { logger } from "azurajs/logger"`** as a class instance: it is a **function**. For **`.info()`**-style methods, use **`new Logger()`** or the **`log`** shortcuts from **`azurajs/logger`**.

```typescript
import { logger, log } from "azurajs/logger";

logger("success", "Connected.");
log.info("Also fine");
```

## 7. Decorators and controllers

v3 uses **`applyDecorators`** from **`azurajs/decorators`** where you compose metadata. Prefer **`app.register(MyController)`** for **`@Controller`** classes. Details: [Controllers](/docs/en/v3/concepts/controllers), [Decorators](/docs/en/v3/concepts/decorators).

## 8. Plugins and middleware

- **`app.plugin(factory())`** registers a **plugin** (framework plugin signature).
- **`app.use(...)`** accepts middleware and compatible handlers; see [Middleware](/docs/en/v3/concepts/middleware).

In v2 many behaviors lived in standalone helpers; in v3 the official list is under **`azurajs/plugins`** (**16** plugins including **`SwaggerPlugin`**). Overview: [Plugins](/docs/en/v3/features/plugins).

## 9. Modular imports (cheat sheet)

| Need | Import from |
| :--- | :--- |
| Server | `azurajs` or `azurajs/core` |
| Plugins | `azurajs/plugins` |
| Route / param decorators | `azurajs/decorators` |
| **`applyDecorators`** | `azurajs/decorators` |
| **`v`**, DTO, Schema validators | `azurajs/validators` |
| Callable **`logger`**, **`configureLogger`** | `azurajs/logger` |
| **`Logger` class** | `azurajs/utils` |
| Swagger decorators / OpenAPI helpers | `azurajs/swagger` |
| Legacy infra-style imports | `azurajs/infra` |

Full list: [Modular imports](/docs/en/v3/advanced/modular-imports).

## 10. Breaking changes summary (v2 → v3)

| Area | Change |
| :--- | :--- |
| Core class | `AzuraClient` → **`AzuraServer`** |
| Swagger | `setup*` functions → **`SwaggerPlugin`**; defaults **`/openapi.json`** and **`/docs`** |
| DTO errors | Prefer **`DtoValidationError`** where applicable |
| Logger | Callable **`logger`** from **`azurajs/logger`**; **`Logger`** class is under **`utils`** |
| npm | New `exports` subpaths — see [package reference](/docs/en/v3/reference/package-and-exports) |

## 11. Common errors after upgrade

| Symptom | Likely cause | Fix |
| :--- | :--- | :--- |
| **`Cannot read properties of undefined (reading 'object')`** on **`v`** | Wrong or missing **`v`** export | Upgrade **`azurajs`**; use **`import { v } from "azurajs/validators"`**. |
| **`Logger` has no call signatures** | **`logger(...)`** with class typing | Import **`logger`** from **`azurajs/logger`**, or **`new Logger()`** from **`azurajs/utils`**. |
| Swagger blank or spec 404 | Wrong **spec** URL | Match **`specPath`** in **`SwaggerPlugin`** (default **`/openapi.json`**). |
| Duplicate **`ValidationError`** types | Mixing DTO with **`v`** | **`DtoValidationError`** for **DTOValidator**; **`ValidationError`** for **`v`**. |
| Subpath types not resolving | Old **`moduleResolution`** | Use **`"Bundler"`** or **`"NodeNext"`** in `tsconfig.json`. |
