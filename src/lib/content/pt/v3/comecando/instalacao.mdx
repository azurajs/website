# Instalação

Este guia vai orientá-lo na instalação do AzuraJS v3 e na configuração de um projeto TypeScript do zero.

## Pré-requisitos

Antes de começar, certifique-se de ter:

- **Node.js 18+** ou **Bun 1.0+** — AzuraJS v3 requer um runtime moderno.
- **TypeScript 5.0+** — Necessário para suporte a decorators e os recursos mais recentes da linguagem.
- **Um gerenciador de pacotes** — npm, yarn, pnpm ou bun.

<Callout type="warning" title="Versão do Node.js">
  AzuraJS v3 utiliza APIs nativas do Node.js (`node:http`, `node:fs`, `node:crypto`, etc.) e requer **Node.js 18 ou posterior**. Versões anteriores não são suportadas.
</Callout>

## Configuração do TypeScript

Crie um `tsconfig.json` na raiz do seu projeto com as seguintes configurações recomendadas:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "Node16",
    "moduleResolution": "Node16",
    "lib": ["ES2022"],
    "outDir": "dist",
    "rootDir": "src",
    "strict": true,
    "esModuleInterop": true,
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true
  },
  "include": ["src"],
  "exclude": ["node_modules", "dist"]
}
```

<Callout type="warning" title="Decorators">
  As flags `experimentalDecorators` e `emitDecoratorMetadata` são **obrigatórias** se você deseja utilizar controllers baseados em decorators (`@Controller`, `@Get`, `@Post`, etc.). Se você usar apenas a API funcional estilo Express (`app.get()`, `app.post()`), essas flags podem ser omitidas.
</Callout>

## Configuração do Package.json

Certifique-se de que seu `package.json` inclua o campo `"type": "module"` para suporte a ESM:

```json
{
  "name": "my-azura-app",
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "dev": "tsx watch src/main.ts",
    "build": "tsc",
    "start": "node dist/main.js"
  },
  "dependencies": {
    "azurajs": "^3.0.0"
  },
  "devDependencies": {
    "@types/node": "^22.0.0",
    "tsx": "^4.0.0",
    "typescript": "^5.4.0"
  }
}
```

<Callout type="info" title="tsx para desenvolvimento">
  Recomendamos o **tsx** como executor de desenvolvimento. Ele executa TypeScript diretamente sem uma etapa de build separada, suporta modo watch e lida com ESM/CJS sem problemas.
</Callout>

## Estrutura de Projeto Recomendada

Aqui está o layout de diretórios recomendado para um projeto AzuraJS v3:

```
my-azura-app/
├── src/
│   ├── controllers/          # Controllers baseados em decorators
│   │   ├── UserController.ts
│   │   └── AuthController.ts
│   ├── middleware/            # Funções de middleware customizadas
│   │   └── auth.ts
│   ├── services/             # Lógica de negócios
│   │   └── UserService.ts
│   ├── validators/           # Schemas DTO e validação
│   │   └── UserDTO.ts
│   └── main.ts               # Ponto de entrada da aplicação
├── azura.config.ts            # Arquivo de configuração do AzuraJS (opcional)
├── tsconfig.json
├── package.json
└── README.md
```

### Detalhamento dos Diretórios

| Diretório | Finalidade |
|---|---|
| `src/controllers/` | Classes decoradas com `@Controller` que definem rotas e handlers. |
| `src/middleware/` | Funções de middleware customizadas para autenticação, logging, validação, etc. |
| `src/services/` | Lógica de negócios desacoplada das preocupações HTTP. |
| `src/validators/` | Schemas DTO usados com `DTOValidator` ou `SchemaValidator`. |
| `src/main.ts` | Ponto de entrada — cria o `AzuraServer`, registra plugins e controllers, inicia a escuta. |
| `azura.config.ts` | Arquivo de configuração opcional carregado automaticamente pelo framework. |

## Importações Modulares

AzuraJS v3 suporta importações modulares com tree-shaking. Em vez de importar tudo do pacote raiz, você pode importar de sub-módulos específicos:

```typescript
// Importação raiz — tudo disponível
import { AzuraServer, CORSPlugin, Controller, Get } from "azurajs";

// Ou importações modulares — apenas o que você precisa
import { AzuraServer } from "azurajs/core";
import { CORSPlugin, HelmetPlugin } from "azurajs/plugins";
import { Controller, Get, Post, Body } from "azurajs/decorators";
import { HttpError, Logger } from "azurajs/utils";
import { ConfigModule } from "azurajs/config";
import { LoggingMiddleware } from "azurajs/middleware";
import type { AzuraConfig, AzuraRequest, AzuraResponse } from "azurajs/types";
```

| Módulo | Conteúdo |
|---|---|
| `azurajs/core` | `AzuraServer`, `Router` |
| `azurajs/plugins` | Todos os 15 plugins integrados, `signJWT`, `verifyJWT` |
| `azurajs/decorators` | `Controller`, `Get`, `Post`, `Put`, `Delete`, `Patch`, `Body`, `Query`, `Param`, `Headers`, `Cookies`, `Ip`, `Session`, `Req`, `Res`, `Meta`, `UseMiddleware`, `applyDecorators` |
| `azurajs/types` | `AzuraConfig`, `AzuraRequest`, `AzuraResponse`, `HttpStatus` e todos os tipos de opções de plugins |
| `azurajs/utils` | `HttpError`, `Logger`, `DTOValidator`, `SchemaValidator`, utilitários de parsing |
| `azurajs/config` | `ConfigModule` |
| `azurajs/middleware` | `LoggingMiddleware` |

## Suporte a Runtimes

<Tabs items={["Node.js", "Bun"]}>
<Tab value="Node.js">
AzuraJS roda nativamente em Node.js 18+. Nenhum polyfill ou adaptador necessário.

```bash
# Desenvolvimento
npx tsx watch src/main.ts

# Produção
node dist/main.js
```
</Tab>
<Tab value="Bun">
AzuraJS roda em Bun sem nenhuma configuração extra. O suporte nativo a TypeScript do Bun significa que você pode pular a etapa de build completamente.

```bash
# Desenvolvimento & Produção
bun run src/main.ts

# Com modo watch
bun --watch src/main.ts
```
</Tab>
</Tabs>

## Verificar a Instalação

Crie um arquivo `src/main.ts` mínimo para verificar se tudo está funcionando:

```typescript
import { AzuraServer } from "azurajs";

const app = new AzuraServer({
  server: { port: 3000 },
});

app.get("/", (req, res) => {
  res.json({ status: "ok", framework: "AzuraJS v3.0.0" });
});

app.listen(3000, () => {
  console.log("AzuraJS is running on http://localhost:3000");
});
```

Execute:

<Tabs items={["tsx", "bun"]}>
<Tab value="tsx">
```bash
npx tsx src/main.ts
```
</Tab>
<Tab value="bun">
```bash
bun src/main.ts
```
</Tab>
</Tabs>

Acesse `http://localhost:3000` — você deverá ver:

```json
{ "status": "ok", "framework": "AzuraJS v3.0.0" }
```
