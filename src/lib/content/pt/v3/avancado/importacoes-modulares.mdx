# Importações Modulares 📦

O AzuraJS v3 publica **todos os subcaminhos** no campo `exports` do `package.json` (mais `typesVersions` para o TypeScript). Cada entrada aponta para um módulo com tipos dedicados — ideal para **tree-shaking** e imports mínimos.

Índice alinhado ao npm: [Pacote npm e exports](/docs/pt/v3/referencia/pacote-e-exports).

## Mapa completo de subcaminhos

| Subcaminho | Descrição | Exports principais |
|---|---|---|
| `azurajs` | Entrada completa | `AzuraServer`, `Router`, decorators, plugins, utils, config, middleware, types |
| `azurajs/core` | Núcleo | `AzuraServer`, `Router` |
| `azurajs/plugins` | Plugins oficiais (incl. Swagger) | `CORSPlugin`, `RateLimitPlugin`, `HelmetPlugin`, `JWTPlugin`, `SessionPlugin`, `CompressionPlugin`, `StaticPlugin`, `ETagPlugin`, `RequestIdPlugin`, `TimeoutPlugin`, `HealthCheckPlugin`, `CircuitBreakerPlugin`, `SSEPlugin`, `SSEManager`, `ProxyPlugin`, `MultipartPlugin`, `SwaggerPlugin`, `signJWT`, `verifyJWT`, … |
| `azurajs/decorators` | Decorators | `Controller`, `Get`, `Post`, …, `Body`, `Query`, `Param`, `UseMiddleware`, `applyDecorators`, `Meta`, … |
| `azurajs/types` | Tipos e constantes HTTP | `AzuraRequest`, `AzuraResponse`, `HttpStatus`, `HttpStatusText`, opções de plugins, … |
| `azurajs/utils` | Utilitários | `Logger`, `HttpError`, `v`, `DTOValidator`, `SchemaValidator`, parsers, cookies, `resolveIp` |
| `azurajs/validators` | Só validação | `v`, `ValidationError`, `Infer`, `DTOValidator`, `SchemaValidator`, … |
| `azurajs/logger` | Logger chamável (v2) | `logger`, `log`, `configureLogger`, `colors` |
| `azurajs/config` | Configuração | `ConfigModule` |
| `azurajs/middleware` | Middlewares embutidos | `LoggingMiddleware` |
| `azurajs/router` | Só roteador | `Router` |
| `azurajs/cookies` | Cookies (helpers) | `serializeCookie`, `clearCookieHeader` |
| `azurajs/http-error` | Só erros HTTP | `HttpError` |
| `azurajs/cors` | CORS mínimo | `CORSPlugin`, `CORSOptions` |
| `azurajs/rate-limit` | Rate limit mínimo | `RateLimitPlugin`, `RateLimitOptions` |
| `azurajs/infra` | Compatível com imports “infra” da v2 | `ConfigModule`, `resolveIp` |
| `azurajs/swagger` | OpenAPI + UI | `SwaggerPlugin`, `buildOpenApiDocument`, `ApiTags`, `ApiOperation`, `renderSwaggerUiPage`, … |
| `azurajs/swagger-ui` | Ficheiro estático | `swagger-ui-modern.html` (não é módulo JS) |

## Importação Completa vs. Modular

### Importação Completa (não recomendado para produção)

```typescript
import {
  AzuraServer,
  Controller,
  Get,
  CORSPlugin,
  HttpError,
  ConfigModule,
} from "azurajs";
```

Funciona perfeitamente, mas o bundler precisa analisar o módulo raiz inteiro para determinar o que descartar.

### Importação Modular (recomendado)

```typescript
import { AzuraServer } from "azurajs/core";
import { Controller, Get, Body, Param } from "azurajs/decorators";
import { CORSPlugin, RateLimitPlugin } from "azurajs/plugins";
import { HttpError } from "azurajs/utils";
import type { AzuraRequest, AzuraResponse } from "azurajs/types";
```

Cada `import` resolve para um arquivo de entrada menor. O bundler (esbuild, Rollup, webpack) faz tree-shaking apenas no subconjunto importado.

## Detalhamento por Subcaminho

### `azurajs/core`

```typescript
import { AzuraServer, Router } from "azurajs/core";

const app = new AzuraServer({ server: { port: 3000 } });

const apiRouter = new Router();
apiRouter.add("GET", "/health", (req, res) => res.json({ ok: true }));
```

`AzuraServer` é a classe principal que gerencia o ciclo de vida HTTP. `Router` é o roteador radix-tree de alta performance utilizado internamente.

### `azurajs/plugins`

```typescript
import { CORSPlugin, HelmetPlugin, JWTPlugin, signJWT } from "azurajs/plugins";

app.plugin(CORSPlugin({ origin: "*" }));
app.plugin(HelmetPlugin());
app.plugin(JWTPlugin({ secret: process.env.JWT_SECRET! }));

const token = signJWT({ userId: "123" }, process.env.JWT_SECRET!);
```

Cada plugin é uma factory que retorna um `PluginHandler`. A lista completa inclui **16** plugins oficiais (incluindo **Swagger**): CORS, Rate Limit, Helmet, JWT, Session, Compression, Static, ETag, Request ID, Timeout, Health Check, Circuit Breaker, SSE, Proxy, Multipart e Swagger.

### `azurajs/decorators`

```typescript
import {
  Controller,
  Get,
  Post,
  Body,
  Param,
  UseMiddleware,
  Meta,
} from "azurajs/decorators";

@Controller("/users")
class UserController {
  @Get("/:id")
  findOne(@Param("id") id: string) {
    return { id };
  }

  @Post()
  @Meta({ roles: ["admin"] })
  create(@Body() data: unknown) {
    return { created: true, data };
  }
}
```

Decorators disponíveis: **Classe** — `Controller`, `UseMiddleware`; **Método** — `Get`, `Post`, `Put`, `Delete`, `Patch`, `Head`, `Options`, `Meta`; **Parâmetro** — `Body`, `Query`, `Param`, `Headers`, `Cookies`, `Req`, `Res`, `NextFunc`, `Ip`, `Session`.

### `azurajs/types`

```typescript
import type {
  AzuraRequest,
  AzuraResponse,
  MiddlewareHandler,
  ErrorHandler,
  HttpMethod,
  AzuraConfig,
  CORSOptions,
} from "azurajs/types";
import { HttpStatus, HttpStatusText } from "azurajs/types";

const notFound: number = HttpStatus.NOT_FOUND; // 404
const text: string = HttpStatusText[404]; // "Not Found"
```

Este subcaminho exporta **apenas tipos** (com exceção de `HttpStatus` e `HttpStatusText`), então seu impacto no bundle é praticamente zero.

### `azurajs/utils`

```typescript
import { HttpError, Logger, DTOValidator } from "azurajs/utils";
import type { Schema, ValidationResult } from "azurajs/validators";

throw HttpError.badRequest("Campo 'email' é obrigatório");

const logger = new Logger({ level: "debug", colors: true });
logger.info("Servidor inicializado");

const schema: Schema = {
  name: { type: "string", required: true },
  age: { type: "number", min: 0 },
};
const result: ValidationResult = DTOValidator.validate(schema, body);
```

Inclui também funções de parsing (`parseBody`, `parseQueryString`, `parseCookies`, `parseUrl`) e gerenciamento de cookies (`serializeCookie`, `clearCookieHeader`).

### `azurajs/config`

```typescript
import { ConfigModule } from "azurajs/config";

const config = await ConfigModule.load();

ConfigModule.merge({
  server: { port: 8080 },
  logging: { level: "debug" },
});
```

O `ConfigModule` é um singleton que carrega automaticamente de `azura.config.ts`, `.js`, `.mjs`, `.json`, `.yaml` ou `.yml`.

### `azurajs/middleware`

```typescript
import { LoggingMiddleware } from "azurajs/middleware";

app.use(LoggingMiddleware());
```

Middlewares embutidos prontos para uso. Atualmente expõe o `LoggingMiddleware` para log automático de requisições.

## Tree-Shaking na Prática

Considere um projeto que usa apenas o servidor, CORS e dois decorators:

```typescript
import { AzuraServer } from "azurajs/core";
import { Controller, Get } from "azurajs/decorators";
import { CORSPlugin } from "azurajs/plugins";
```

Neste cenário, o bundler **não inclui** no output final: os outros plugins, decorators não usados, `Logger`, `HttpError`, validadores extra, `ConfigModule`, `LoggingMiddleware`, etc.

<Callout type="tip" title="Dica de Performance">
  Para builds de produção com `tsup`, `esbuild` ou `rollup`, importações modulares podem reduzir o tamanho do bundle em **40-60%** comparado com `import * from "azurajs"`.
</Callout>

## Compatibilidade com CommonJS e ESM

Todos os subcaminhos suportam ambos os sistemas de módulos:

```typescript
// ESM
import { AzuraServer } from "azurajs/core";

// CommonJS
const { AzuraServer } = require("azurajs/core");
```

O campo `exports` no `package.json` usa condições `import` (ESM) e `require` (CJS), com `types` dedicado para cada um.
