# Logger

O AzuraJS expõe **duas** formas de registar mensagens:

1. **`logger` chamável** e atalhos **`log`** em **`azurajs/logger`** — igual à **v2** (`logger("success", "…")`, ícones e cores).
2. **Classe `Logger`** em **`azurajs/utils`** — métodos **`.info()`**, **`.error()`**, usada pelo **`AzuraServer`** e middleware.

<Callout type="warning" title="Não confundas os dois `logger`">
  **`import { logger } from "azurajs/logger"`** é uma **função**. **`logger.info(...)`** dá erro de tipo. Usa **`log.info(...)`** do mesmo módulo, ou **`new Logger()`** de **`azurajs/utils`**.
</Callout>

## Logger chamável (`azurajs/logger`)

Para arranque, Redis, PostgreSQL, scripts:

```typescript
import { logger, log, configureLogger } from "azurajs/logger";

logger("log", "A ligar ao Redis…");
logger("success", "Redis ligado.");
logger("fatal", "Falha na ligação.");

log.info("À escuta na porta 3000");
log.error("Algo correu mal");
```

### Níveis (primeiro argumento)

| Nível | Uso |
| :--- | :--- |
| `debug`, `trace` | Diagnóstico |
| `info`, `log` | Mensagens normais |
| `success` | Sucesso |
| `warn` | Avisos |
| `error`, `fatal` | Saída via **`console.error`** |

### `configureLogger`

```typescript
configureLogger({
  prefix: "MinhaAPI",
  showTimestamp: true,
  timestampFormat: "iso",
});
```

### `colors`

`import { colors } from "azurajs/logger"` — **`colors.hex`**, **`colors.ansi`**, etc.

## Classe `Logger` (`azurajs/utils`)

```typescript
import { Logger } from "azurajs/utils";

const log = new Logger({
  level: "debug",
  prefix: "minha-app",
  timestamp: true,
  colors: true,
});

log.debug("Cache miss");
log.info("Servidor na porta 3000");
log.warn("Endpoint depreciado");
log.error("Erro:", err.message);
```

### Opções do construtor

| Opção | Tipo | Padrão |
| :--- | :--- | :--- |
| `level` | `debug` … `silent` | `"info"` |
| `prefix` | `string` | `"azura"` |
| `timestamp` | `boolean` | `true` |
| `colors` | `boolean` | auto (TTY) |

### Níveis (classe)

Ordem de severidade: `debug` → `info` → `warn` → `error` → `silent`.

### `request`, `banner`, `setLevel`

- **`request(method, path, status, duraçãoBigInt)`** — linha HTTP (uso interno; ver abaixo).  
- **`banner(port, host)`** — banner após **`listen`**.  
- **`setLevel("warn")`** — alterar em runtime.

### Cores por método e status

Quando **`requestLogging`** está ativo, o servidor usa o **`Logger`** interno com cores por método HTTP (GET verde, POST azul, etc.) e por faixa de status (2xx verde, 5xx vermelho, …).

## Configuração no `AzuraServer`

```typescript
const app = new AzuraServer({
  logging: {
    enabled: true,
    level: "info",
    requestLogging: true,
    colors: true,
    timestamp: true,
  },
});
```

Com **`enabled: false`**, o nível passa a **`silent`**.

## `getLogger()` no servidor

```typescript
const app = new AzuraServer();
const serverLogger = app.getLogger();
serverLogger.info("Mensagem com o mesmo Logger da framework");
```

## LoggingMiddleware

```typescript
import { LoggingMiddleware } from "azurajs/middleware";

app.plugin(LoggingMiddleware());
```

Mede o tempo com **`process.hrtime.bigint()`**, define **`X-Response-Time`** e regista via logger do servidor.

## Quando usar o quê

| Caso | API |
| :--- | :--- |
| Paridade com tutoriais **v2** | **`azurajs/logger`** — **`logger("success", …)`** |
| Preferes **`.info()`** num único import | **`log`** de **`azurajs/logger`** ou **`new Logger()`** |
| Prefixo por módulo | **`new Logger({ prefix: "db" })`** |
